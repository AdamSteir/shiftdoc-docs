Test Name: All add-ons - sections present in JSON output for QA Manager
Feature: Add-ons Analysis
User Role: QA Manager
Traceability: @SHIFT-15
Description: Verifies that when a QA Manager runs a full analysis with all add-ons enabled, the downloaded JSON output contains all expected sections with meaningful content.
This Test Verifies:
Verification StepWhat It Checks
Documentation and analysis appears in output panel The analyze page produces visible results after uploading a BDD test file with all add-ons selected
JSON output has no unresolved values The downloaded report is fully populated with no placeholder or missing data
JSON contains top-level key 'technical_review' The technical review section is present in the report
JSON contains top-level key 'gap_analysis' The gap analysis section is present in the report
JSON contains top-level key 'annotation_summary' The annotation summary section is present in the report
JSON contains top-level key 'risk_assessment' The risk assessment section is present in the report
JSON contains top-level key 'recommendations' The recommendations section is present in the report
'technical_review' has non-empty 'test_quality' The technical review section includes actual test quality findings
'gap_analysis' has non-empty 'coverage_gaps' The gap analysis section identifies real coverage gaps
'annotation_summary' has non-empty 'annotation_health' The annotation summary section provides meaningful annotation health data
'risk_assessment' has non-empty 'scenario_risk_ratings' The risk assessment section includes risk ratings for scenarios
'recommendations' has non-empty 'actions' The recommendations section provides actionable next steps
