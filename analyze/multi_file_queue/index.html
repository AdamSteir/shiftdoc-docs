Test Name: Logged-in user can enable multi-file mode
Feature: Multi-file mode — queue operations
Traceability: @SHIFT-48
Description: Verifies that enabling multi-file mode shows the file queue and hides the single-file input area.
This Test Verifies:
Verification StepWhat It Checks
When I enable multi-file mode The user can switch the page into multi-file mode
Then the file queue is visible The file queue appears after enabling multi-file mode
And the single-file input area is not visible The single-file input is hidden when multi-file mode is active
Test Name: Switching to multi-file mode clears the single-file input
Feature: Multi-file mode — queue operations
Traceability: @SHIFT-48
Description: Verifies that any content typed in the single-file input is cleared when the user switches to multi-file mode.
This Test Verifies:
Verification StepWhat It Checks
Given I have typed content in the input area A baseline of pre-existing content in the single-file input is established
When I enable multi-file mode The user switches to multi-file mode
Then the input area content is cleared Previously entered content is removed when switching modes, preventing stale data carry-over
Test Name: Switching back to single-file mode clears the queue
Feature: Multi-file mode — queue operations
Traceability: @SHIFT-48
Description: Verifies that switching back to single-file mode hides the queue (including any queued files) and restores the single-file input area.
This Test Verifies:
Verification StepWhat It Checks
When I enable multi-file mode and add a file to the queue A file is staged in the queue while in multi-file mode
And I disable multi-file mode The user switches back to single-file mode
Then the file queue is not visible The queue is hidden after returning to single-file mode
And the single-file input area is visible The single-file input area is restored when multi-file mode is turned off
Test Name: Files can be added and removed from the queue
Feature: Multi-file mode — queue operations
Traceability: @SHIFT-48
Description: Verifies that a user can add a file to the queue and then remove it, leaving the queue empty.
This Test Verifies:
Verification StepWhat It Checks
When I add a fixture file to the queue A file can be successfully added to the multi-file queue
Then the queue shows 1 file The queue count accurately reflects the number of staged files
When I remove the file from the queue A file can be removed from the queue by the user
Then the queue is empty The queue correctly shows as empty after removal
Test Name: Files can be added to the queue via drag-and-drop
Feature: Multi-file mode — queue operations
Traceability: @SHIFT-48
Description: Verifies that a user can drag and drop a file onto the queue drop zone and it appears in the queue with a success status.
This Test Verifies:
Verification StepWhat It Checks
When I drop a fixture file onto the queue drop zone Drag-and-drop is a supported method for adding files to the queue
Then the queue shows 1 file The dropped file is registered in the queue
And the file row shows an ok badge The file is accepted and marked with a positive status indicator
Test Name: Adding a file with a duplicate filename shows a toast and does not add it to the queue
Feature: Multi-file mode — queue operations
Traceability: @SHIFT-48
Description: Verifies that attempting to add a file that is already in the queue shows a warning message and prevents a duplicate entry.
This Test Verifies:
Verification StepWhat It Checks
When I add the same fixture file to the queue a second time The system detects a duplicate filename being added to the queue
Then I see a warning toast containing 'is already in the queue' The user is notified with a clear warning message about the duplicate
And the queue shows 1 file The duplicate file is not added, keeping the queue count accurate
