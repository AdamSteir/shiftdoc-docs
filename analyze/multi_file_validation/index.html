Multi File Validation

Framework: BDD Generated: 2026-06-23 Model: claude-sonnet-4-6
Test Name: Adding more than the maximum files shows a warning toast and rejects the file
Feature: Multi-file mode — file limits and validation
Traceability: @SHIFT-50
Description: Verifies that when the file queue is already at its maximum capacity, attempting to add another file is blocked and the user receives a warning notification.
This Test Verifies:
Verification StepWhat It Checks
Fill queue to maximum then add one more The system prevents exceeding the maximum file limit by rejecting the additional file
Warning toast is displayed The user is clearly notified that the file limit has been reached
Queue file count does not increase The rejected file is not added to the queue, keeping the count at the maximum
Test Name: Adding two files when one slot remains toasts and accepts only the first
Feature: Multi-file mode — file limits and validation
Traceability: @SHIFT-50
Description: Verifies that when only one slot remains in the queue and two files are added simultaneously, only the first file is accepted, the second is rejected, and the user receives a warning notification.
This Test Verifies:
Verification StepWhat It Checks
Fill queue to 19 files then add 2 at once The system partially accepts a batch upload, taking only as many files as there are remaining slots
Warning toast is displayed The user is notified that the file limit was reached and not all files could be added
Queue shows 20 files The queue reached the maximum of 20 and the overflow file was discarded
